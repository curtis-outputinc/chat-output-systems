# Session logs

At the end of every conversation (or before context likely gets compacted), write a summary to `sessions/YYYY-MM-DD-<short-slug>.md`. If a file for today's session already exists and the work is continuous, append to it instead of creating a new one.

Each summary should cover:
- **Goal** — what the user asked for in their own framing
- **Changes** — files touched, what was added/edited/removed (with paths)
- **Decisions** — non-obvious choices made and why
- **Follow-ups** — anything left open, broken, or deferred
- **Notes** — surprises, dead ends, things future-me should know

Keep it terse. This is a log, not a report. The goal is that opening any session file gives enough context to pick up where we left off without re-reading the whole transcript.
