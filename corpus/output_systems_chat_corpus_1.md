# SECTION 7: PRIVACY, DATA, AND COMPLIANCE

- Output Systems builds every intelligent chat system with privacy and data compliance built in from the start, not bolted on afterward
- Output Systems collects only the data required for the function the system performs
- Data collected for one purpose is not used for another without disclosure
- Customer data is stored securely within approved infrastructure
- Access controls are configured so only authorized team members can access conversation data
- The system is designed with appropriate guardrails that protect both the business and its customers
- Two categories of data are kept completely separate: anonymized insight data and appointment/lead data
- Anonymized insight data: the questions visitors ask and the topics they explore, with no names, contact details, or personal information attached
- Appointment and lead data: name, phone number, email, preferred contact method, and explicit consent, collected only when a visitor chooses to book or be contacted
- Conversation content, personal narratives, and case-specific details are never written to any database
- The system is specifically designed to prevent sensitive personal information from being stored
- No third parties have access to visitor data
- Privacy regulations change. As part of Monthly System Management, Output Systems conducts a standing review of any significant changes to privacy and data compliance requirements across all relevant jurisdictions.
- If a regulation changes in a way that affects how a system collects, stores, or displays customer information, Output Systems updates the system and communicates the change to the client.

## Regulatory Frameworks Output Systems Systems Are Built Around

- PIPEDA: Canada — federal private sector privacy law
- Quebec Law 25: Canada — enhanced provincial privacy requirements
- CASL: Canada — anti-spam law covering commercial electronic messages
- GDPR: European Union and UK — applies when customers include EU or UK residents
- CCPA and CPRA: California — applies when customers include California residents
- Other applicable US state privacy laws are considered on a client-by-client basis

## Responsible AI Use

- Every system is built with guardrails
- The system defines what it can answer, what it should avoid, what information it can collect, when it should escalate to a human, and how it protects the business's reputation
